# Scenario 7: High‑Profile User / Board Member Sensitive Access

## Goal
Protect high‑impact personas from impersonation during sensitive actions.

## Functional Steps to Test
1. A senior executive accesses sensitive information.
2. The action is flagged as high impact.
3. Verified ID is required.
4. Face Check is enforced.
5. Verification results are logged.
6. Access proceeds only after successful validation.

## Expected Outcomes
- Identity assurance aligns with role criticality.
- Verification is friction‑appropriate and auditable.
