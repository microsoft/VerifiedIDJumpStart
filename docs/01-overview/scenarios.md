# Typical Scenarios Where Face Check Applies

| Scenario | Description |
|--------|-------------|
| High‑risk user onboarding | Prove a new user is the legitimate identity holder |
| Privileged access elevation | Confirm operator presence before sensitive actions |
| Account recovery | Prevent social‑engineering‑based account resets |
| Remote workforce verification | Validate identity without physical presence |
| Regulated access events | Meet assurance requirements for compliance‑bound systems |
| Step‑up verification for anomalies | Dynamically respond to risky sign‑in activity |
